## 3. DDPG 算法详解 (Deep Deterministic Policy Gradient)

> [!summary] 一句话总结 DDPG
> 
> 如果说 PPO 是个性格保守、每次只走一小步的谨慎派；那 DDPG 就是个**带了错题本（经验回放池）**、**只认死理（确定性策略）**的实干家。它专门用来解决输出连续数值的控制问题。

### 3.1 核心直观理解

回忆一下前面的 [[前置极简知识&PPO 算法详解 (Proximal Policy Optimization)]]：它输出的是概率（比如“70% 向左，30% 向右”），然后去抽签决定动作。

但在现实中，很多机器人的控制是**连续的**（[[Continuous Action Space|连续动作空间]]）。比如你想让机械臂抓杯子，你不能说“70% 用 10 牛顿的力，30% 用 20 牛顿的力”，你需要它直接给出一个精确的数字：“精准发力 15.3 牛顿”。

- **DDPG 的做法**：它放弃了摇骰子（概率分布），直接让 Actor（演员）输出一个**确定性的唯一动作**（[[Deterministic Policy|确定性策略]]）。就像你掷飞镖，不估算落点概率，而是直接指着靶心说：“我就往这儿扎！”
    

### 3.2 核心思想（大白话）

DDPG 依然沿用了 [[Actor-Critic|演员-评论家架构]]，但它为了让自己学习得更快、更稳，偷偷带了“三件法宝”：

1. **绝对自信的 Actor**：不再输出概率，给定一个状态 $s$，直接输出一个具体的动作 $a$。
    
2. **[[Replay Buffer|经验回放池]]（错题本）**：因为每次输出的动作是确定的，DDPG 必须通过在动作里加点“噪音”（瞎抖一抖）来探索新世界。它把所有的经历（无论好坏）都存进一个大池子里，每次训练时随机抓一把出来复习。这也让它成为一种极度节约数据的 [[Off-policy|异策略]] 算法。
    
3. **[[Target Network|目标网络]]（影子替身）**：在训练 Critic（评论家）时，如果标准总变，模型就会精神分裂（今天觉得 10 分算好，明天觉得 20 分才算好）。DDPG 复制了一套更新非常缓慢的“影子网络”来作为打分的参考标准，防止目标剧烈震荡。
    

### 3.3 关键公式（极简+大白话解释）

为了不让你被吓到，我们把 DDPG 的网络分成两组：**现实网络**（正在训练的，性子急）和 **影子网络**（作为参考的，性子慢）。

**公式 1：评论家的目标（目标 Q 值）**

$$y = r + \gamma Q_{target}(s', \pi_{target}(s'))$$

> [!note] 大白话拆解：
> 
> - $y$：**及格线（目标分数）**。这是现实中的评论家想要努力预测准的最终总分。
>     
> - $r$：**眼前实实在在拿到的奖励**。
>     
> - $\gamma$：**折扣因子**（比如 0.99），表示未来的奖励要打个折。
>     
> - $Q_{target}(\dots)$ 和 $\pi_{target}(\dots)$：**影子评论家** 和 **影子演员**。
>     
> 
> **总结**：现实评论家怎么学打分？它要对比“自己的打分”和“及格线”。及格线是怎么来的呢？是**眼前的真实奖励**，加上**影子评论家**根据**影子演员**的下一步动作给出的预估分。

**公式 2：演员的优化目标**

$$J \approx Q_{local}(s, \pi_{local}(s))$$

> [!note] 大白话拆解：
> 
> 现实演员 $\pi_{local}$ 要怎么进步？很简单，它就去问现实评论家 $Q_{local}$：“我怎么动，你才能给我打最高的分？” 演员的目标，就是让评论家给出的 $Q$ 值最大化。

**公式 3：[[Soft Update|软更新]]（影子网络的进化）**

$$\theta_{target} = \tau \theta_{local} + (1-\tau) \theta_{target}$$

> [!note] 大白话拆解：
> 
> $\tau$ 是一个极小的数（比如 0.005）。每次现实网络（$\theta_{local}$）学完一点东西，影子网络（$\theta_{target}$）只吸收它 0.5% 的新知识，保留自己 99.5% 的老知识。这就叫**软更新**，保证了影子网络的变化像乌龟一样慢，从而维持打分标准的稳定。

### 3.4 算法执行流程

1. **探索世界**：演员根据当前状态 $s$ 做出一个绝对动作 $a$，为了探索，我们在动作上加一点随机噪音（比如手稍微抖一下）。
    
2. **记录错题**：把这一步的 $(s, a, r, s')$ 存入经验回放池。
    
3. **随机抽题**：从回放池里随机抓取一小批数据用于训练（打破数据之间的关联性）。
    
4. **训练评论家**：用影子网络计算出“及格线”，然后让现实评论家去逼近这个及格线（计算误差并修正）。
    
5. **训练演员**：演员向现实评论家讨教，调整自己的动作输出，努力让评论家打出更高的分数。
    
6. **缓慢同步**：用软更新（公式 3），让影子网络慢慢地向现实网络靠拢。
    

### 3.5 伪代码（简洁版）

Python

```
# DDPG 极简伪代码逻辑
初始化 现实演员、现实评论家
复制生成 影子演员、影子评论家 (参数完全一样)
初始化 经验回放池

for 每一回合 in range(最大回合数):
    获取初始状态 s
    
    for 每一小步 in range(最大步数):
        # 1. 带噪音的行动
        确定性动作 a = 现实演员(s) + 探索噪音
        执行 a，得到奖励 r 和新状态 s'
        
        # 2. 存入错题本
        经验回放池.存入(s, a, r, s')
        状态 s = 新状态 s'
        
        # 3. 如果错题本攒够了数据，开始复习
        if 经验回放池.数据量 > 批次大小:
            抽出一个 Batch 的数据
            
            # 4. 更新评论家 (向影子网络对齐)
            影子动作 = 影子演员(s')
            及格线 = r + 折扣因子 * 影子评论家(s', 影子动作)
            评论家误差 = MSE(现实评论家(s,a), 及格线)
            优化(现实评论家, 评论家误差)
            
            # 5. 更新演员 (以评论家的评分为导向)
            演员评分 = 现实评论家(s, 现实演员(s))
            演员误差 = - Mean(演员评分) # 加负号是因为我们要最大化评分
            优化(现实演员, 演员误差)
            
            # 6. 软更新影子替身
            软更新(影子评论家, 现实评论家, 混合率=0.005)
            软更新(影子演员, 现实演员, 混合率=0.005)
```

### 3.6 优缺点 & 适用场景

> [!important] DDPG 简历

- **优点**：
    
    - **数据极其节约**：有经验回放池，过去的数据可以反复学（比 PPO 省数据得多）。
        
    - **专攻连续控制**：非常适合输出具体数值的任务（比如输出油门深浅 0~100，转向角度 -90~90）。
        
- **缺点**：
    
    - **过度自信（高估偏差）**：这是 DDPG 最大的致命伤！评论家经常给自己的动作打过高的分（吹牛），导致演员盲目自信，最后模型崩溃。
        
    - **调参困难**：相比于 PPO 的“万金油”，DDPG 比较傲娇，超参数稍微不对可能就完全学不会。
        
- **适用场景**：
    
    - 机器狗走路、机械臂抓取、自动驾驶底盘控制等连续动作问题，且在模拟环境中获取数据比较慢/贵的场景。
