# 强化学习核心算法：[[PPO]] /[[DDPG]] / [[TD3]] 新手笔记

## 1. 前置极简知识（新手必读）

在正式学习算法前，我们需要统一下“黑话”。整个 [[Reinforcement Learning|强化学习]]（RL）的核心，就是一个试错的过程，就像“训练小🐱”。

> [!info] 核心概念字典
> 
> - **[[Agent|智能体]]**：你训练的 AI 模型（比如：那只正在学握手的小狗）。
>     
> - **[[Environment|环境]]**：AI 所处的整个世界（比如：你、狗绳、狗零食、周围的声音）。
>     
> - **[[State|状态]] ($s$)**：当前环境的样子（比如：小狗听到“握手”指令）。
>     
> - **[[Action|动作]] ($a$)**：AI 决定的行为（比如：小狗抬起左爪）。
>     
> - **[[Reward|奖励]] ($r$)**：环境给 AI 的反馈（比如：给了小狗一块肉，奖励 +1；小狗咬人，打一顿，奖励 -1）。
>     
> - **[[Policy|策略]] ($\pi$)**：AI 脑子里的“行动指南”。决定了在状态 $s$ 下，选择动作 $a$ 的概率。
>     
> - **[[Value Function|价值函数]] ($V$)**：用来评估某个状态“有多好”。不仅看眼前奖励，还要看长远未来能拿多少奖励。
>

## 2. PPO 算法详解 (Proximal Policy Optimization)

> [!summary] 一句话总结 PPO
> 
> PPO 就像是“摸着石头过河”，每次只敢往前试探一小步。它通过限制策略更新的幅度，保证了学习过程的极度稳定。目前爆火的 ChatGPT 背后的 RLHF，用的核心算法就是 PPO！

### 2.1 核心直观理解

想象你正在学骑自行车（训练 [[Policy|策略]]）。

- **传统算法的毛病**：你发现稍微往左倾斜一点能走得更远，于是下一次你**猛地**往左打死方向盘，结果直接摔倒（策略更新步子太大，导致模型崩溃）。
    
- **[[PPO]] 的做法**：你发现往左倾斜有用，于是你告诉自己：“下次往左倾斜一点点就好，**绝对不能超过规定的幅度**”。这样虽然学得不一定是最快的，但**最稳**，绝不会轻易摔倒。
    

### 2.2 核心思想（大白话）

PPO 是基于 [[Actor-Critic|演员-评论家架构]] 的算法。它有两个大脑：

1. **Actor（演员）**：负责根据当前状态做出动作（骑车的你）。
    
2. **Critic（评论家）**：负责在旁边打分，判断这个动作好不好（旁边的教练）。
    

PPO 的**核心思想**是：使用 [[Importance Sampling|重要性采样]] 让模型可以稍微重复利用一下刚才收集的数据，同时引入一个 **[[Clipping|截断机制]]**，强制规定：**新策略和老策略的区别不能太大**。

### 2.3 关键公式（极简+大白话解释）

PPO 只有这一个你必须懂的核心公式：Actor 的目标函数（也就是演员要优化的目标）。

$$L^{CLIP} = \min \left( \frac{\pi_{new}(a|s)}{\pi_{old}(a|s)} A, \text{clip}\left(\frac{\pi_{new}(a|s)}{\pi_{old}(a|s)}, 1-\epsilon, 1+\epsilon\right) A \right)$$

> [!note] 公式大白话拆解（不要怕，我们一点点看）：
> 
> - $\frac{\pi_{new}(a|s)}{\pi_{old}(a|s)}$：**新老策略比率**。算的是新策略做这个动作的概率，除以老策略做这个动作的概率。如果大于 1，说明新策略更喜欢做这个动作。
>     
> - $A$：**[[Advantage Function|优势函数]]**。代表这个动作比平均水平**好多少**。如果 $A>0$，说明动作好；如果 $A<0$，说明动作烂。
>     
> - $\text{clip}(\dots, 1-\epsilon, 1+\epsilon)$：**核心保命机制（截断）**。$\epsilon$ 通常是一个很小的数（比如 0.2）。这个函数的意思是：不管你新老策略比率有多大或多小，我都强行把它**卡**在 0.8 到 1.2 之间。
>     
> - $\min(\dots)$：取两者中的最小值。这是为了防止模型在优势巨大时“过度自信”，盲目更新。
>     

**大白话总结公式：**

如果一个动作好（$A>0$），就提高做这个动作的概率，但**最多提高 20%**；如果一个动作烂（$A<0$），就降低概率，但**最多降低 20%**。绝不贪心！

### 2.4 算法执行流程

1. **收集经验**：让当前的 Actor 在环境里玩几把，记录下状态、动作、奖励。
    
2. **教练打分**：Critic 根据记录的数据，计算出每个动作的优势 $A$（到底比瞎玩的平均水平好多少）。
    
3. **演员小步更新**：Actor 根据上面的 PPO 截断公式，稍微调整自己的策略，确保新老策略差异不大。
    
4. **教练自我提升**：Critic 根据真实拿到的奖励，更新自己的打分能力，争取下次打分更准。
    
5. **循环**：重复以上步骤，直到小狗（Agent）完全学会技能。
    

### 2.5 伪代码（简洁版）

Python

```
# PPO 极简伪代码逻辑
for 迭代次数 in range(最大迭代数):
    # 1. 收集数据
    老策略数据 = 演员在环境里互动(步数=N) 
    
    # 2. 计算优势
    优势A = 评论家给每个动作打分(老策略数据)
    
    # 3. 更新模型
    for 小批量 in range(更新次数):
        # 限制更新幅度 (Clipping 机制)
        新老策略比 = 计算比例(新策略概率, 老策略概率)
        限制后的比率 = Clip(新老策略比, 0.8, 1.2)
        
        # 优化 Actor (演员)
        演员损失 = - Min(新老策略比 * 优势A, 限制后的比率 * 优势A)
        更新演员网络(演员损失)
        
        # 优化 Critic (评论家)
        评论家损失 = (预测的分数 - 真实获得的分数)^2
        更新评论家网络(评论家损失)
```

### 2.6 优缺点 & 适用场景

> [!important] PPO 简历

- **优点**：
    
    - **超级稳定**：不容易崩溃，调参非常友好（号称 RL 界的“万金油”）。
        
    - **适用性广**：既能做离散动作（比如下棋的上下左右），也能做连续动作（比如控制方向盘的具体角度）。
        
- **缺点**：
    
    - **样本效率一般**：它是 [[On-policy|同策略]] 算法，每次更新策略后，之前收集的老数据就作废了，比较吃环境交互数据。
        
- **适用场景**：
    
    - 环境模拟成本低、可以快速生成大量数据的情况（比如在游戏引擎里训练、做大语言模型微调）。
        
    - 不知道选什么算法时，**首选 PPO 准没错**。
